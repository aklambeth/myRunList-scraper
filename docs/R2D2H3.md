## Extraction Strategy
 
The endpoint returns a raw HTML fragment (no `<html>` or `<body>` wrapper):
 
```html
<thead class="hashTableHeader">
  <tr class="hashTableHeaderRow">
    <th colspan="5" class="hashTableHeaderTitle">Next Twenty Runs</th>
  </tr>
  <tr class="hashTableHeaderRow">
    <th class="hashTableHeaderCell columnFormatClass0">Run No</th>
    <th class="hashTableHeaderCell columnFormatClass1">Run Date</th>
    <th class="hashTableHeaderCell columnFormatClass2">Location</th>
    <th class="hashTableHeaderCell columnFormatClass3">GridRef/Postcode</th>
    <th class="hashTableHeaderCell columnFormatClass4">Notes</th>
  </tr>
</thead>
<tbody class="hashTableBody">
  <tr class="hashTableBodyRowOdd"> ... </tr>
  <tr class="hashTableBodyRowEven"> ... </tr>
  ...
</tbody>
```
 
Each data row has exactly five `<td>` cells in fixed column order:
 
| Index | Column | Example value |
|-------|--------|---------------|
| 0 | Run No | `1406` |
| 1 | Run Date | `18 Jun 2026` |
| 2 | Location | `White Hill car park, Kingsclere.` |
| 3 | GridRef/Postcode | `w3w.eyelash.pony.lamenting` |
| 4 | Notes (hares) | `Top Dog, Headley Hound and Hotlegs` |
 
Parse with BeautifulSoup: select all `<tr>` elements whose class contains
`hashTableBodyRow`, then extract the five `<td>` text values from each.
 
```python
from bs4 import BeautifulSoup
 
def extract_rows(html: str) -> list[list[str]]:
    soup = BeautifulSoup(html, "html.parser")
    rows = soup.find_all("tr", class_=lambda c: c and "hashTableBodyRow" in c)
    return [
        [td.get_text(strip=True) for td in row.find_all("td")]
        for row in rows
        if len(row.find_all("td")) == 5
    ]
```

## GridRef/Postcode Column Parsing
 
The `GridRef/Postcode` column (index 3) is the most varied field and may contain
any of the following:
 
| Pattern | Example | Mapping |
|---------|---------|---------|
| W3W with `w3w.` prefix | `w3w.eyelash.pony.lamenting` | `location.w3s` = `eyelash.pony.lamenting` |
| OS grid ref + postcode | `489 344/SO23 7NU` | `location.osgrid` + `location.postcode` |
| Postcode only | `SO23 7NU` | `location.postcode` |
| OS grid ref only | `SU489344` | `location.osgrid` |
| `TBC` or empty | `TBC` | Skip — omit location fields |
 
```python
import re
 
def parse_gridref_postcode(cell: str) -> dict:
    result = {}
    cell = cell.strip()
 
    if not cell or cell.upper() == "TBC":
        return result
 
    # What Three Words — prefixed with "w3w."
    if cell.lower().startswith("w3w."):
        slug = cell[4:].strip("/")
        if re.match(r'^[a-z]+\.[a-z]+\.[a-z]+$', slug):
            result["w3s"] = slug
        return result
 
    # OS grid ref + postcode combined: "489 344/SO23 7NU"
    combined = re.match(
        r'^([A-Z]{0,2}\d[\d\s]{3,})\s*/\s*([A-Z]{1,2}\d{1,2}\s*\d[A-Z]{2})$',
        cell, re.IGNORECASE
    )
    if combined:
        osgrid_raw = combined.group(1).strip()
        result["postcode"] = combined.group(2).strip().upper()
        # Normalise OS grid: strip spaces, uppercase
        result["osgrid"] = osgrid_raw.replace(" ", "").upper()
        return result
 
    # Postcode only
    pc = re.match(r'^([A-Z]{1,2}\d{1,2}\s*\d[A-Z]{2})$', cell, re.IGNORECASE)
    if pc:
        result["postcode"] = pc.group(1).strip().upper()
        return result
 
    # OS grid ref only (e.g. "SU489344" or "SU 489 344")
    osgrid = re.match(r'^[A-Z]{2}\d{4,}$', cell.replace(" ", ""), re.IGNORECASE)
    if osgrid:
        result["osgrid"] = cell.replace(" ", "").upper()
        return result
 
    return result
```
 
**Note on the `489 344/SO23 7NU` format**: the grid ref portion lacks a two-letter
prefix (e.g. `SU`). Store as-is — the schema accepts the bare numeric form and the
prefix can be inferred from the postcode area if needed downstream. Do not fabricate
a prefix.

## Notes Column — Hare Parsing
 
The Notes column (index 4) contains hare names, sometimes with additional
annotations. Examples:
 
```
"Top Dog, Headley Hound and Hotlegs"
"NBP and Careless"
"Cruella and Cooee - 15 pound curry night Pre-book with Cruella"
"Hares Needed"
"Delhi Boy"
```
 
Split on `,` and ` and ` to extract hare names, then strip each token. If
additional annotation text follows a ` - ` after the last hare name, split there
first and use the left portion for hare extraction; move the right portion to
`notes`.
 
```python
import re
 
def parse_notes(cell: str) -> dict:
    result = {}
    cell = cell.strip()
 
    if not cell or cell.lower() in ("tbc", "hares needed", ""):
        return result
 
    # Split annotation from hare names on first " - "
    parts = cell.split(" - ", maxsplit=1)
    hare_part = parts[0].strip()
    if len(parts) > 1:
        result["notes"] = parts[1].strip()
 
    # Split hare names on ", " and " and "
    names = re.split(r',\s*|\s+and\s+', hare_part)
    names = [n.strip() for n in names if n.strip()]
 
    if names:
        result["hares"] = names
 
    return result
```

## Date Parsing
 
Run dates are in the format `DD Mon YYYY` (e.g. `18 Jun 2026`). Parse with
`datetime.strptime`:
 
```python
from datetime import datetime
 
def parse_date(cell: str) -> str:
    return datetime.strptime(cell.strip(), "%d %b %Y").strftime("%Y-%m-%d")
```
 
No time is provided in the data — omit the `time` field from output.

## Schema Field Mapping
 
| Schema field | Source | Transform |
|---|---|---|
| `name` | Static | `"R2D2 Hash House Harriers"` |
| `runno` | Column 0 | `int()` |
| `date` | Column 1 | `strptime("%d %b %Y")` → `"%Y-%m-%d"` |
| `time` | — | Not available |
| `location.name` | Column 2 | Direct, unless `TBC` |
| `location.postcode` | Column 3 | See GridRef/Postcode parsing |
| `location.osgrid` | Column 3 | See GridRef/Postcode parsing |
| `location.w3s` | Column 3 | See GridRef/Postcode parsing |
| `location.address` | — | Not available |
| `location.lat` | — | Not available |
| `location.lng` | — | Not available |
| `hares` | Column 4 | Split on `,` / ` and ` |
| `notes` | Column 4 | Annotation following ` - ` in Notes cell |
| `oninn` | — | Not available |
| `website` | Static | `"https://www.r2d2h3.com"` |
 
**TBC handling**: if the Location cell is `TBC`, omit `location.name`. If the
GridRef/Postcode cell is `TBC`, omit all location sub-fields. An empty `location`
object is valid output.

## Failure Mode Guidance
 
| Condition | Action |
|-----------|--------|
| HTTP non-200 from ASPX endpoint | `ScraperException(FailureMode.TRANSIENT)` |
| Response contains no `hashTableBodyRow` rows | `ScraperException(FailureMode.FATAL)` — endpoint structure changed |
| Row has fewer than 5 `<td>` cells | Skip row, log warning |
| Run No cell is not a valid integer | Skip row, log warning |
| Date cell cannot be parsed | Skip row, log warning |
